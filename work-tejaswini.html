Dijkstra's Algorithm For finding shortest path from University

Why Dijkstra's Algorithm?

Dijkstra's algorithm plays a crucial role in finding the shortest path from a fixed source node to a specified target location in a weighted, undirected graph.

This code uses Dijkstra's algorithm on an adjacency list representation of a weighted graph to compute the shortest path and distance from a fixed source (UniversityMainGate) to a user-specified target location, enabling efficient navigation in the given network.

How Does it Work?

  • Graph Representation: The university's locations and connections are represented as a weighted graph using an adjacency list.
  • Shortest Path Calculation: Dijkstra's algorithm is employed to compute the shortest path from the fixed source (UniversityMainGate) to the target location, minimizing the total distance.
  • Path Reconstruction: Using the array, the algorithm reconstructs the optimal path from the source to the target, skipping unnecessary nodes.

Benefits:

  • The approach handles arbitrary node connections and edge weights, allowing for a general-purpose navigation system.
  • It works with any type of graph connections and distances, making it adaptable to various navigation needs.
  • The algorithm efficiently computes the shortest paths using the priority queue.

Prim's Algorithm For Connecting Parking Lots

Why Prim's Algorithm?

Prim's algorithm to find the minimum spanning tree (MST) of a graph. The MST is a subset of the edges that connects all the nodes with the minimum possible total weight, without forming any cycle

How Does it Work?

  • Graph Representation: The parking lots (nodes) and roads (edges) are represented as a weighted graph using an adjacency list, where each edge has a distance (weight) between two parking lots.
  • Minimum Spanning Tree (MST) Calculation: Prim's algorithm is employed to compute the minimum spanning tree (MST) by selecting the least costly edge at each step, ensuring all parking lots are connected with the minimal total distance.
  • Cost Calculation: Using a priority queue (min-heap), the algorithm adds the cheapest edge to the MST, updating the total cost and ensuring the optimal connection of all parking lots without forming cycles.

Benefits:

  • Prim's algorithm ensures that all parking lots are connected with the minimum possible total distance, avoiding unnecessary paths and reducing travel costs.
  • Prim's algorithm is easy to implement and understand, making it an accessible choice for solving problems involving optimal connectivity in graphs like parking lot networks
  • Easy to implement and understand, especially for common navigation problems.

Quick Sort for Exam HallAllocation Based on USN

Why Quick Sort?

Quick Sort is a highly efficient, comparison-based sorting algorithm that uses the divide-and-conquer approach to sort elements.

How Does it Work?

  • Sorting the USNs: Quick Sort is used to sort the students' USNs in ascending order, ensuring that students are allocated to exam halls in a consistent and predictable sequence.
  • Efficient Allocation: By sorting the students, the allocation process becomes systematic, where students with smaller USNs are allocated first, optimizing the allocation process across multiple exam halls.
  • Order Preservation: Quick Sort ensures that the students are assigned to halls in a sorted order, maintaining clarity and consistency in the allocation process.

Benefits:

  • Quick Sort provides a structured and predictable order for allocating students to exam halls, ensuring efficient and orderly allocation.
  • By sorting students in \(O(n \log n)\) time, Quick Sort enables faster and more efficient allocation, especially for large numbers of students.
  • Quick Sort's efficiency makes it suitable for handling large datasets of students, enabling the program to scale well as the number of students or exam halls increases.
